一、架构

1、传统的集群架构 和 分布式架构 区别

（1）、集群架构

a、缺点：
- 耦合度太高了
- 增加了团队的合作成本
- 不能够去灵活的部署

（2）分布式架构

a、优点：
- 项目拆分成多个模块，耦合度降低
- 单点运行，团队合作效率高
-可以灵活部署

b、缺点：
-需要去额外的开发，让各个模块之间能够通信。


二、Git

代码的版本控制管理系统

1、Git可以干什么？

- 防止代码丢失
- 远程同步代码
- 团队合作利器
- 代码还原
- 记录代码版本

2、安装Git

- 直接到https://git-scm.com/下载安装即可

3、Git的常用操作命令

- git init 初始化git仓库
- git status 查看状态
- git add 将文件添加到git仓库的暂存区
- git commit 将添加到暂存区的文件提交到git仓库
- git log 查看日志
- git branch 查看当前的分支
- git branch A 创建一个叫做“A”的分支
- git checkout A 切换分支到“A”分支
- git branch -d A 删除“A”分支

- git clone 克隆代码到本地
- git config 配置信息
- git push 将本地代码推送到github等服务器上去
- git pull 将远程的代码更新下来
- git merge 合并分支

4、Github

-全球最大的社交编程网站！

5、Github有什么用？

- 代码托管
- 多人协作
- 个人博客
- 个人简历

6、使用github
a、
- star 收藏
- fork 复制
- repository 仓库
- watch 观察
-Gist 代码片段

b、ssh授权
使用git bash工具运行：
- 生成秘钥：ssh-keygen -t rsa
- 复制公钥到github网站上
- 验证是否授权成功：ssh -T git@github.com


三、maven

1、项目管理
- 提供了一套完整的构建生命周期的框架

2、整合工具
- 自动的完成一些工程的基础构建配置

3、maven的安装
（1）配置好JDK的环境变量
（2）在https://maven.apache.org/ 网站下载maven
（3）配置一下maven的环境变量
（4）运行 mvn --version 检验是否安装成功

4、maven配置文件

（1）配置镜像：

----
		<mirror>
			<id>alimaven</id>
			<mirrorOf>central</mirrorOf>
			<name>aliyun maven mirror</name>
			<url>http://maven.aliyun.com/nexus/content/groups/public/</url>
		<mirror>
----

（2）配置本地仓库

----
		<localRepository>E:/mavenpository</localRepository>
----


5、maven特性

（1）依赖管理
- jar包依赖
- 工程间的依赖
- 继承
- 聚合

（2）工程类型
- war包的项目
- jar包的项目
- pom工程
